# Tolucan Times LA — Project Brief

## What this is
A static HTML/CSS/JS newspaper website for **The Tolucan Times**, a real local newspaper based in Toluca Lake, LA (est. 1937). Eric is building/maintaining this for his friend who runs the paper. No npm, no build tools, no frameworks — pure HTML/CSS/JS only, deployable as a static folder.

## Architecture

### Component system
All pages use `partloader.js` (jQuery `$.get`) to inject shared HTML components into placeholder divs. Components live in `html/`. Only fetches a component if its placeholder `id` exists on the page.

| Placeholder ID | Component file | Used on |
|---|---|---|
| `nav-ph` | `html/navComp.html` | Every page |
| `footer-ph` | `html/footerComp.html` | Every page |
| `slide-ph` | `html/slideComp.html` | index.html only |
| `publication-ph` | `html/publicationComp.html` | index.html only |
| `homepageboxnav-ph` | `html/homepageboxnavComp.html` | index.html only |

### Article system
- Articles live at root as `article_01.html`, `article_02.html`, etc.
- Article metadata + content stored in `data/articles.json`
- `html/homepageboxnavComp.html` is **auto-regenerated** by the admin panel — don't hand-edit it
- Article HTML files are also **auto-generated** by the admin panel

### Data files
- **`data/articles.json`** — array of all article objects (see format below)
- **`data/config.json`** — site-wide config: ad settings, nav slogan, `featuredIds` array (5 slots), `issues` array (ordered list of issue names)

### Admin panel (`admin.html`) — two-column layout
All logic is self-contained in `admin.html` (no separate JS file). Pure JS, connects to GitHub API directly.

**Left panel** — article management:
- Login with a GitHub Personal Access Token (repo scope), stored in localStorage
- Articles organized into **issue folders** (collapsible) — each folder groups articles by their `issue` field
- Folders: named issues (e.g. "April 2026"), an **Unassigned** catch-all, and an **Archive** section
- Drag handle on each article row — drag to assign to a featured slot (right panel)
- Click **Edit** to open the article editor in the left panel (replaces list view)
- **+ New Article** button, **+ Add Issue** input to create new issue folders
- Articles can be archived (hidden from site, restorable)
- Dirty indicator (orange dot ●) on unsaved articles; star badge (★) on featured ones

**Right panel** — site controls:
- **Featured Articles**: 5 drag-drop slots. Slot 1 = large main feature with overlay text; Slots 2–5 = hover thumbnails in the carousel. Drag an article from the left to fill a slot. Empty slots auto-fill with most-recent articles on publish.
- **Nav Slogan**: optional text that replaces the date in the nav bar (e.g. "Happy 4th of July!")
- **Advertisement**: image URL + link URL + alt text for the homepage sidebar ad box

**Publish flow** (single "Publish to Site" button in fixed bottom bar):
1. Writes/updates each dirty article's HTML file (`article_NN.html`)
2. Deletes HTML files for any deleted articles
3. Writes updated `data/articles.json`
4. Regenerates `html/homepageboxnavComp.html` (the featured carousel + More Stories grid)
5. Writes updated `data/config.json` (featured slots, slogan, ad)
All steps commit directly to the GitHub repo; GitHub Pages deploys in ~1 min.

**Changes are local until published** — saving in the editor just marks things dirty locally; nothing hits GitHub until "Publish to Site".

### Article editor (left panel)
Block-based: articles built from ordered blocks (text or image). Fields:
- Headline, Author, Category Tag (dropdown), Date (display string), Issue (assigns to a folder)
- Card Image URL (with live preview), Homepage Excerpt (with char counter)
- Content blocks: text blocks (blank line = new paragraph) and image blocks (URL, alt text, caption)
- Blocks can be reordered (↑↓) or deleted

### Homepage carousel (`homepageboxnavComp.html`)
Generated by `buildHomepageComp()` in admin.html:
- **YC-carousel**: large main feature (left) + 4 thumbnail hover items (right). Clicking or hovering a thumb switches the main panel. Clicking navigates to the article.
- **More Stories grid**: up to 6 articles not in the featured 5, shown as cards below the carousel
- **Explore section**: static links to Photo Gallery, Horoscopes, About Us
- Weather widget (open-meteo API, no key required) and Ad box both rendered client-side from `data/config.json`

## Design intent
- **Nav stays exactly as-is** — Eric loves it, do not touch it
- **Homepage aesthetic:** dense Yahoo/newspaper front page circa 2010 — cream background (`#faf8f3`), dashed column rules, section headers, tight typography
- **Fonts:** Fraunces (headlines), Manrope (UI/meta), Bookish (body text) — all loaded locally from `fonts/` via `css/style.css` @font-face
- **Color palette:** dodgerblue (#1E90FF) primary, orange-brown gradient (#de8c35 → #c54a2b) for accents

## Key decisions made
- `partloader.js` uses check-before-fetch to avoid unnecessary requests on non-homepage pages
- Swiper is initialized only in `html/slideComp.html` — not in `index.html` (removed duplicate)
- Gallery images use `loading="lazy"` — browser loads them as user scrolls
- All admin JS is inline in `admin.html` — no separate admin JS file
- `homepageboxnavComp.html` is fully regenerated on every publish (do not hand-edit)
- Article template at `article_template.html` for reference (not used by admin)
- Weather uses open-meteo (no API key needed), hardcoded to Toluca Lake lat/lon
- Ad and weather both loaded client-side from `data/config.json` at page load (not baked into HTML)
